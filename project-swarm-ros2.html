ROS2 Multi-Robot Swarm Navigation with Nav2

ROS2 Jazzy · Gazebo Harmonic · Nav2 · TurtleBot3 · Arizona State University · Jan 2026

A coordinated 4-robot swarm navigation system where TurtleBot3 Waffle robots navigate autonomously in a shared Gazebo environment using ROS2 Jazzy and Nav2. An operator specifies a single goal position via RViz, and all four robots navigate to coordinated positions while maintaining their relative formation geometry.

Demo

How It Works

The system uses a delta movement strategy — each robot moves by the same displacement vector as the leader robot, ensuring each travels its own unobstructed corridor without inter-robot conflicts:

  • Sequential Dispatch: The leader navigates first; followers dispatch after leader SUCCESS, preventing mid-navigation collisions.
  • Auto-Recovery Spiral Search: When a goal is blocked, the system automatically searches nearby positions in a spiral pattern (up to 5 retries, 8 candidates per ring at 0.2 m step).
  • Laser Scan Filtering: Each robot filters teammates from its laser scan, preventing robots from treating each other as static obstacles.
  • Goal Cooldown: Prevents duplicate RViz messages from cancelling in-progress navigation.
  • Map Bounds Clamping: Goals are automatically clamped to safe navigable area.

System Architecture

The swarm controller node handles delta movement calculation, sequential dispatch, and spiral search auto-recovery. Each of the 4 robots runs its own independent Nav2 stack with NavFn (Dijkstra) path planner and Regulated Pure Pursuit local controller:

  • robot1 — Leader (y = 0.0)
  • robot2 — Follower (y = +1.0)
  • robot3 — Follower (y = −1.2)
  • robot4 — Follower rear (y = 0.0)

Results

Achieved an overall 85–90% goal success rate across all tested scenarios in a ~6.3 × 5.9 m map with a 3×3 cylindrical obstacle grid. Zero mid-navigation collisions across 6 goal trials. Blocked goals were resolved within 0.3–0.5 m of target via the spiral search recovery.
